Mask-Predict: Parallel Decoding of Conditional Masked Language Models

​ 本文介绍了 conditional masked language models (CMLMs) 和mask-predict解码算法 ，该算法利用它们的并行性在恒定的解码迭代次数中生成文本。

长度预测

​ 在encoder中加入一个特殊的 $LENGTH$ token ， 类似于 BRET 中的CLS ，encoder在训练后预测长度N，输入到decoder

​ 在实验中，我们选择了可能性最高的 $l$ 个$LENGTH$ , 并对同一输入进行不同长度的解码，最终选取 平均对数概率最大 的结果作为输出

CMLM

​ conditional masked language model (CMLM)

​ 给定原句 X 和 目标句子的部分tokens Yobs，来预测目标句子的剩余部分 Ymask

这是基于 Ymask中的词 在给定 X 和 Yobs时 是相互条件独立的强假设上的。

​ 句子的长度 N = |Ymask| + |Yobs| 即两者长度之和。

MASK

​ 在encoder端得到目标序列长度的预测 N 后， 我们可以定义2个变量。

  • target sequence (y1,y2,,,,yN)

  • 每个位置的可能性(p1,p2,,,,,pN)

对于第1次迭代 ( t = 0)，我们mask所有的token。对于后面的迭代，我们将概率得分p最低的 n 个 tokens 进行 mask 。

Predict

​ 在对句子进行mask后，我们通过源句子 X 和 没有被mask的Yobs 来预测 Ymask， Ymask选择最大可能的token。并更新这一位置上概率分数p。对于没有遮掩的位置Yobs，它们的概率值保持不变。

Analysis

  • Why Are Multiple Iterations Necessary?
    • 为什么需要多次的迭代
    • 多次$mask-predict$迭代通过允许模型对部分输入进行条件化来缓解多模态问题
      • 输入的句子中的 Yobs 为之后的预测提供了一些限制条件，缓解了 multi-modality 的问题
  • Do Longer Sequences Need More Iterations?
    • 长句子是否需要更多次的迭代
    • 实验显示确实有提示，但即使迭代次数较少，也口译达到较好的效果
  • Do More Length Candidates Help?
    • 选取更多可能的长度是否会使结果更好
    • 有提高，但太多的话反而会下降
      • 和beam serch不一样，有太多候选长度话，可能会选到 概率较小的长度(比如一个很短的长度) ， 但在这个长度下，其输出有一个很高的对数平均概率，导致错误的结果
  • Is Model Distillation Necessary?
    • Distillation 是否有必要?
    • 实验结果显示很有必要
