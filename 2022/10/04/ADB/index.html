ADB的使用

ADB

追觅扫地机器人

  1. 插入录音程序
    adb push 刷机程序位置 目标位置eg : adb push d:/Desktop/speech /data/
  2. 链接到设备（清扫记录查看ip地址,端口为4567）
    adb connect ip地址:4567eg : adb connect 192.168.3.30:4567
  3. 查看是否连接到设备
    adb devices
  4. 进入shell
    adb shell
  5. 进入之前插入的刷机程序位置
    cd /data/speech
  6. 杀死可能存在就进程
    killall monitor_speech
    killall vos_cleaner
  7. 运行录音程序
    ./vos_cleaner
  8. 结束运行
    ctrl + c放后台运行的话 killall vos_cleaner
  9. 拉出录音(需要退出shell)
    adb pull /data/speech_auth/origin.pcm .可能遇到的问题:无法录音：将speech/app_cfg.json中的debug_origin 以及 debug_beforming 设为1

一数投影仪

CMD窗口下

  1. 运行以下命令，先执行关闭风扇的脚本
  2. 给root权限
    adb root
  3. 链接设备(usb)
    adb remount
  4. 进入adb shell
    adb shell
  5. 开始录音
    tinycap /sdcard/SNR.wav -D 0 -d 6 -c 4 -r 16000 -b 16
  6. 拷出音频(退出adb shell)
    adb pull /sdcard/SNR.wav
  • 无法执行关闭风扇的脚本时运行下面的代码

adb shell “echo 456 > /sys/class/gpio/export”
adb shell “echo out > sys/class/gpio/gpio456/direction”
adb shell “echo 0> sys/class/gpio/gpio456/value”
//使能手动调整风扇
adb shell “echo 0 > sys/devices/platform/customer-fan/autofan”
//风扇的四个等级，值越大，风扇噪声越小
adb shell “echo 0 100> sys/devices/platform/customer-fan/duty”

  • 老版本apk 录音运行代码
    tinycap /sdcard/xxx.wav -D 0 -d 7 -c 4 -r 16000 -b 16

R328调试命令

  • 抓日志：logread -f /tmp/.lastlog

  • 查SN：cat /mnt/UDISK/msmart_store_info.txt

  • 跳过生产模式：touch /etc/tinatest_ok；

  • 看duilite库的版本：ll /mnt/app/libs

  • 开启语音测试功能：touch /mnt/app/.asr_test

  • 退出语音测试功能：cd /mnt/app/，

  • 然后rm -f .asr_test

  • 关闭语音测试功能：rm /mnt/app/.asr_test

  • 然后重启 保存音频的指令：touch /mnt/app/.debug_save_asr_audio

    ​ touch /mnt/app/.save_whole_rec;

  • 录音指令：arecord -D hw:audiocodec -c 3 -r 16000 -f S16_LE /mnt/UDISK/xiaoshengshi.wav

  • 如果录音机被占用无法录音，重新命名指令：mv /mnt/app/deamon /mnt/app/deamon1

  • 然后重启后即可录音

  • 录音使用完后，命名改回来就行：mv /mnt/app/deamon1 /mnt/app/deamon

  • 播放指令：aplay /tmp/test.wav

  • 音量调节：amixer -D hw:audiocodec cset name=’LINEOUT volume’ 21 最大值21

  • 只看唤醒&识别的日志：logread -f | grep “xiao mei|rec"“

  • 音频push路径 /mnt/UDISK
